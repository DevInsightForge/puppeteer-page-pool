# PuppeteerPagePool

`PuppeteerPagePool` is a callback-based Chromium page pool for rendering-heavy PuppeteerSharp workloads such as:

- HTML to PDF generation
- HTML to image generation
- repeated server-side page rendering
- authenticated report rendering against existing web apps

The package is intentionally pool-centric. It manages browser startup, page reuse, reset, replacement, host integration, and health checks while exposing a lease-scoped `ILeasedPage` surface for the work you perform inside the pool callback.

## Install

```bash
dotnet add package PuppeteerPagePool
```

## Target frameworks

- `net8.0`
- `net10.0`

## Design

The pool keeps ownership of browser pages.

You do not receive a raw `IPage`, and you do not dispose or close pooled pages yourself. Instead, you execute work through `IPagePool`:

```csharp
public interface IPagePool : IAsyncDisposable
{
    ValueTask ExecuteAsync(
        Func<ILeasedPage, CancellationToken, ValueTask> operation,
        CancellationToken cancellationToken = default);

    ValueTask<TResult> ExecuteAsync<TResult>(
        Func<ILeasedPage, CancellationToken, ValueTask<TResult>> operation,
        CancellationToken cancellationToken = default);

    ValueTask<PagePoolHealthSnapshot> GetSnapshotAsync(CancellationToken cancellationToken = default);
}
```

This gives the pool three important guarantees:

- the page object handed to user code is lease-scoped
- the wrapper becomes invalid as soon as the callback ends
- the pool decides whether to reset or replace the underlying page

## Package surface

### `IPagePool`

Use `IPagePool` to:

- execute work with a leased page
- return values from the callback
- inspect pool health and capacity

### `ILeasedPage`

`ILeasedPage` is a thin rendering-oriented wrapper over the underlying page. It supports:

- navigation
- HTML content loading
- waiting for page readiness
- JavaScript evaluation
- viewport and header configuration
- authentication
- PDF generation
- screenshot generation

It does not expose close or dispose operations.

### `PagePoolOptions`

`PagePoolOptions` controls:

- pool size and acquire timeout
- startup warmup and shutdown drain timeout
- reset target and reset navigation behavior
- cookie and storage cleanup on return
- maximum page reuse count
- per-page configuration and pre-lease hooks
- local browser launch options
- remote browser connect options

### Health support

The package includes:

- `PagePoolHealthSnapshot`
- `AddPuppeteerPagePoolHealthCheck()`

## Registration

Register the pool with dependency injection:

```csharp
using Microsoft.Extensions.DependencyInjection;
using PuppeteerPagePool;
using PuppeteerSharp;

var services = new ServiceCollection();

services.AddPuppeteerPagePool(options =>
{
    options.PoolSize = 4;
    options.AcquireTimeout = TimeSpan.FromSeconds(20);
    options.ShutdownTimeout = TimeSpan.FromSeconds(20);
    options.ResetTargetUrl = "about:blank";
    options.LaunchOptions = new LaunchOptions
    {
        Headless = true,
        Timeout = 120_000,
        Args =
        [
            "--disable-dev-shm-usage",
            "--disable-gpu",
            "--no-sandbox"
        ]
    };
});
```

Resolve and use the pool:

```csharp
await using var provider = services.BuildServiceProvider();
var pool = provider.GetRequiredService<IPagePool>();
```

## Quick start

Render a page to PDF:

```csharp
var pdf = await pool.ExecuteAsync(
    async (page, cancellationToken) =>
    {
        await page.SetContentAsync(
            """
            <!doctype html>
            <html>
            <body>
                <h1>Quarterly report</h1>
                <p>Generated by PuppeteerPagePool.</p>
            </body>
            </html>
            """);

        await page.WaitForNetworkIdleAsync();

        return await page.PdfDataAsync(new PdfOptions
        {
            PrintBackground = true
        });
    });
```

Render a page to image:

```csharp
var image = await pool.ExecuteAsync(
    async (page, cancellationToken) =>
    {
        await page.SetViewportAsync(new ViewPortOptions
        {
            Width = 1440,
            Height = 900
        });

        await page.GoToAsync("https://example.com/report");
        await page.WaitForNetworkIdleAsync();

        return await page.ScreenshotDataAsync(new ScreenshotOptions
        {
            FullPage = true
        });
    });
```

## Common usage patterns

### Load HTML directly

```csharp
await pool.ExecuteAsync(
    async (page, cancellationToken) =>
    {
        await page.SetContentAsync("<html><body><h1>Hello</h1></body></html>");
        await page.WaitForFunctionAsync("() => document.readyState === 'complete'");
    });
```

### Render an authenticated page

```csharp
await pool.ExecuteAsync(
    async (page, cancellationToken) =>
    {
        await page.SetExtraHttpHeadersAsync(new Dictionary<string, string>
        {
            ["X-Report-Key"] = "internal-key"
        });

        await page.AuthenticateAsync(new Credentials
        {
            Username = "report-user",
            Password = "secret"
        });

        await page.GoToAsync("https://example.com/reports/monthly");
        await page.WaitForSelectorAsync("#report-root");
    });
```

### Compute values during rendering

```csharp
var result = await pool.ExecuteAsync(
    async (page, cancellationToken) =>
    {
        await page.GoToAsync("https://example.com/report");

        var title = await page.GetTitleAsync();
        var ready = await page.EvaluateExpressionAsync<string>("document.readyState");

        return new
        {
            title,
            ready
        };
    });
```

## Hooks

You can configure each page once when it is created:

```csharp
services.AddPuppeteerPagePool(options =>
{
    options.ResetTargetUrl = "about:blank";
    options.ConfigurePageAsync = async (page, cancellationToken) =>
    {
        await page.SetViewportAsync(new ViewPortOptions
        {
            Width = 1280,
            Height = 720
        });
    };
});
```

You can also run logic before every lease:

```csharp
services.AddPuppeteerPagePool(options =>
{
    options.ResetTargetUrl = "about:blank";
    options.BeforeLeaseAsync = async (page, cancellationToken) =>
    {
        await page.WaitForFunctionAsync("() => document.readyState === 'complete'");
    };
});
```

The hook receives the same lease-scoped `ILeasedPage` type that user callbacks receive, and it becomes invalid once the hook completes.

## Browser configuration

### Launch a local Chromium process

```csharp
services.AddPuppeteerPagePool(options =>
{
    options.ResetTargetUrl = "about:blank";
    options.LaunchOptions = new LaunchOptions
    {
        Headless = true,
        Timeout = 120_000
    };
});
```

`PuppeteerPagePool` always forces locally launched browsers to run headless. It overrides `LaunchOptions.Headless`, `LaunchOptions.HeadlessMode`, and `LaunchOptions.Devtools` so the pool does not launch a visible browser window.

If `LaunchOptions.ExecutablePath` is not supplied:

- `PUPPETEER_EXECUTABLE_PATH` is used when available
- otherwise the package downloads Chromium through `BrowserFetcher`

### Connect to an existing browser

```csharp
services.AddPuppeteerPagePool(options =>
{
    options.ResetTargetUrl = "about:blank";
    options.ConnectOptions = new ConnectOptions
    {
        BrowserWSEndpoint = "ws://localhost:3000/devtools/browser/your-browser-id"
    };
});
```

You cannot set both `LaunchOptions` and `ConnectOptions`.

## Reset and replacement behavior

After each successful callback the pool:

1. navigates the page to `ResetTargetUrl`
2. waits using `ResetWaitConditions`
3. optionally clears cookies
4. optionally clears local and session storage
5. returns the page to the pool

The page is replaced instead of reused when:

- the callback throws
- the page is closed
- the page belongs to an old browser generation
- the page exceeds `MaxPageUses`
- reset fails
- page preparation fails before the lease

If the browser disconnects, the pool rebuilds browser state and repopulates the page channel.

## Health checks

Register the built-in health check:

```csharp
services.AddHealthChecks()
    .AddPuppeteerPagePoolHealthCheck();
```

Health behavior:

- unhealthy when the browser is disconnected
- unhealthy when the browser is unresponsive
- degraded when the pool is not accepting new leases
- healthy when the pool is ready

## Snapshot

Inspect pool state at runtime:

```csharp
var snapshot = await pool.GetSnapshotAsync();
```

`PagePoolHealthSnapshot` includes:

- `PoolSize`
- `AvailablePages`
- `LeasedPages`
- `WaitingRequests`
- `BrowserConnected`
- `AcceptingLeases`

## Exceptions

- `PagePoolAcquireTimeoutException`
  Thrown when no page becomes available before `AcquireTimeout`.
- `PagePoolDisposedException`
  Thrown when work is requested after disposal or after lease acceptance stops.
- `PagePoolUnavailableException`
  Thrown when browser startup, browser connection, executable validation, or rebuild fails.

## Operational guidance

- Keep callbacks short and bounded.
- Return plain values or rendered bytes from callbacks.
- Do not store `ILeasedPage` references beyond the callback.
- Prefer `about:blank` or another very fast internal page as `ResetTargetUrl`.
- Use `ConfigurePageAsync` for stable per-page setup such as viewport defaults.
- Use `BeforeLeaseAsync` for lightweight readiness checks or request headers that must be refreshed.

## What the package does not try to be

`PuppeteerPagePool` is not:

- a workflow engine
- a scraping framework
- a full PuppeteerSharp wrapper
- a browser abstraction that hides PuppeteerSharp option types

It is a pooled page runtime for repeated rendering work.
